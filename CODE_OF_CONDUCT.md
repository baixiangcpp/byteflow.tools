# Code of Conduct

Byteflow aims to be a productive, respectful open-source project for developers who value privacy-first tooling.

## Expected Behavior

- Be respectful and specific in issues, pull requests, reviews, and discussions.
- Assume good intent, but keep technical feedback concrete and evidence-based.
- Welcome contributors regardless of experience level, background, identity, location, or language.
- Keep conversations focused on the project and the change being discussed.
- Respect privacy. Do not post another person's private information, credentials, tokens, or sensitive payloads.

## Unacceptable Behavior

- Harassment, insults, threats, or sustained disruption.
- Sexualized language or imagery.
- Discriminatory comments or exclusionary behavior.
- Publishing private information without explicit permission.
- Repeatedly derailing issues, pull requests, or discussions after maintainers ask for focus.

## Enforcement

Maintainers may edit or remove comments, close discussions, reject contributions, temporarily restrict participation, or permanently block participants when behavior harms the project or community.

If you need to report a conduct issue, contact the project maintainer through GitHub at <https://github.com/baixiangcpp/byteflow.tools>. If the report is sensitive, open a brief issue asking for a private contact path and omit private details from the public thread. Include relevant links, screenshots, and context when possible. Reports will be handled with discretion.
